A Brief History of GLib and JSON-GLib

This page provides a concise overview of the origins and evolution of two important libraries in the world of C programming: GLib and JSON-GLib.

GLib

GLib is a general-purpose utility library that offers essential tools like data structures, threading, and file handling for C programs. It began in the mid-1990s as part of the GTK project, a toolkit for creating graphical user interfaces. Originally developed by Peter Mattis, Spencer Kimball, and Josh MacDonald, GLib was designed to support GTK but quickly grew beyond that role. By the late 1990s, it had become a standalone library, widely used in the GNOME ecosystem and other open-source projects. Its first stable release, version 2.0, came in 2002. Today, GLib is prized for its portability and robust features, forming the backbone of many applications.

JSON-GLib

Built on top of GLib, JSON-GLib is a library focused on handling JSON data—parsing it, generating it, and working with it in C. It appeared in 2007, as JSON rose to prominence as a lightweight data format. Primarily developed by Emmanuele Bassi and contributors from the GNOME community, JSON-GLib integrates with GLib’s GObject system, offering a straightforward API for JSON tasks. Its first stable release, version 0.6, arrived in 2009. It’s now a go-to choice for GNOME applications that need to interact with web APIs or process JSON data.

Current Status

Both libraries remain active and widely used. GLib provides the foundational infrastructure, while JSON-GLib extends its capabilities to meet modern data exchange needs. Together, they’re key components in the open-source software landscape.

Example Usage

Here’s a simple example of how JSON-GLib might be used to parse a JSON string: 

#include <json-glib/json-glib.h>

int main() {
    const char *json = "{\"name\": \"Example\", \"value\": 42}";
    JsonParser *parser = json_parser_new();
    json_parser_load_from_data(parser, json, -1, NULL);
    JsonNode *root = json_parser_get_root(parser);
    g_print("Parsed JSON successfully\n");
    g_object_unref(parser);
    return 0;
}
