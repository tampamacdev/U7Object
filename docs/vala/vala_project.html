Vala Starter Project for WebSocket Endpoint

This Vala project shows you how to create a network service that provides a WebSocket endpoint for establishing two way communication between endpoints. It uses the Vala wrapper libraries for the C libraries GLib and JSON-GLib. Is also uses a custom Vala bridge for the embedded web server Mongoose.

A custom set of classes to mimic macOS standard property list classes (NSString, NSData, NSDate, NSNumber, NSDictionary and NSArray) are included in the Vala starter project. The property list classes also inherit from a parent class that provides convenience methods to encode/decode property lists to/from json data (see Custom Vala Base Class).

The main purpose of this Vala project is to build an API supporting WebSocket communication between remote systems. The class that performs the heavy lifting is U7ThreadLoop. I made a decent attempt to document how to subclass and use U7ThreadLoop, but the real documentation, in my honest opinion, is in the implementation file U7ThreadLoop.vala. The Vala file 'U7WebSocketManager.vala' is how this Vala project interfaces with the C code that manages the Mongoose web server. Look at the C file mongoose_vala_bridge.c to get a feel for the possibilities when embedding a lightweight web server into your application. The Mongoose bridge code lets you set the socket port you want to listen on with the start_websocket_server() function. Read the documentation on Ngrok tunneling, found in the home index. This document shows you how to create an account and host your WebSocket service from anywhere. Finally, follow the instructions for building the Vala project here.

The Vala project builds a product named "websocket_endpoint" in the build/ folder. A documentation that explains the initialization steps found in the main() function can be found here.
